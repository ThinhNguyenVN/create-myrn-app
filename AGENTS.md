## Repository guidelines

- Use normal working branches in the format `feat/<description>`, `fix/<description>`, `issue/<description>`, or `release/<description>`.
- Do **not** create `cursor/*` branches for regular repository work.
- Treat `cursor/*` as platform-managed automation branches only when the hosting environment forces that convention.
- Before opening a pull request, make sure the branch name follows the normal repository rule above whenever you control the branch name.
